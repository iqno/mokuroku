# Mokuroku - Internal Directory Indexer

**Author:** Alen Zejnic
**Purpose:** Local file organization within the Obsidian environment.

## Overview

Mokuroku is a custom, lightweight script developed to automate the organization of local Markdown files. As directory structures grow, maintaining an updated index of files in each directory becomes tedious and prone to human error. Mokuroku solves this by automatically generating and updating a "Map of Content" (index file) within each directory. 

These index files contain a clean, auto-updating list of links to all notes and subdirectories within that specific folder, ensuring that files are never orphaned and remain easily navigable.

## Security & Compliance Statement

This tool was designed from the ground up to comply with strict workplace security policies:

*   **Zero Runtime Dependencies:** The codebase has been stripped of all third-party external runtime libraries. It relies exclusively on the standard Obsidian plugin API.
*   **Fully Local Execution:** All file processing and indexing happens 100% locally on the host machine.
*   **Air-Gapped Operation:** The code contains absolutely zero network calls, API requests, or telemetry tracking. It cannot transmit data outside of the local environment.
*   **Non-Destructive File Modification:** The script only modifies the specific auto-generated link blocks between the defined `%% Mokuroku: ... %%` tags. It respects and preserves all other user-written content in the file.

## Technical Workflow

Mokuroku operates by listening to standard local file events (creation, deletion, and renaming). 

1. **Event Detection:** When a file event is triggered, the script determines which directories are affected using symmetric difference logic to compare the current state against the last known state.
2. **Index Generation:** For the affected directories, it compiles an alphabetically sorted list of the markdown files and subdirectories present.
3. **Link Formatting:** The file paths are formatted into standard markdown links.
4. **File Update:** It locates the index file (default name: `_Index_of_[FolderName].md`), searches for the designated Mokuroku metadata markers, and injects the updated list. If no index file exists, it creates one.

## Configuration & Usage

The script exposes several configurable parameters directly within the UI to customize its behavior:

| Setting | Description |
| :--- | :--- |
| **Index Prefix** | The naming convention for generated files (Default: `_Index_of_`). |
| **List Style** | How links are formatted (Pure Link, Bulleted List, or Checkbox). |
| **Meta Tags** | YAML frontmatter tags automatically appended to index files (e.g., `MOC`) to enable global search queries. |
| **Clean Files** | Strips file paths and `.md` extensions for cleaner readability. |
| **Include/Exclude Rules** | Absolute path definitions to explicitly include or exclude specific directories from the indexing process. |
| **Template Support** | Allows pointing the script to a boilerplate markdown template for scaffolding new index files. |

## Codebase Structure

To facilitate easy auditing and review, the codebase has been flattened and simplified:
*   `main.ts`: Contains the core event listeners, file traversal logic, and settings UI definition.
*   `utils.ts`: Contains pure, stateless utility functions for manipulating markdown content, regex matching, and frontmatter extraction.
*   `types.ts`: Contains all TypeScript interfaces and enums for strict typing.
*   `consts.ts`: Defines the hardcoded marker strings used to identify the autogenerated sections.
